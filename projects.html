Large-Scale Data Analytics Dashboard

This project involved the design and implementation of a large-scale analytics dashboard for the technical production unit, built on top of a Databricks environment containing more than one billion rows of coil production measurements collected over the last three years.

The goal of the dashboard was to enable technical and production teams to run simulations over historical data in order to evaluate whether new customer orders could be fulfilled without risking material loss or compromising production constraints.

Simulation performance was a critical requirement, as users needed to evaluate multiple scenarios in a short period of time. Long execution times would make iterative analysis impractical and directly impact decision-making.

To achieve the required performance and scalability, I led the following technical initiatives:

  • Query optimization: Conducted an in-depth analysis of complex queries involving multiple filters and joins, restructuring them to minimize execution time on large datasets.
  • View materialization: Identified performance bottlenecks and created materialized views to precompute expensive operations, significantly reducing query latency.
  • Dashboard implementation: Designed and implemented the final solution using Databricks Dashboards, enabling interactive simulations over the full dataset.

As a result, the dashboard was able to execute simulations over the entire dataset in under 3 minutes, making it viable for real-time, scenario-based decision-making in a high-stakes industrial environment.

Web Platform Performance Optimization

The application originally took up to 8 seconds to fully load each page, which resulted in frequent user complaints and poor perceived performance. After internal discussions, I was responsible for investigating the root causes and implementing technical improvements.

The platform stack at the time was:

  • Framework: WordPress (PHP)
  • Database: MariaDB
  • Cloud provider: SaveInCloud
  • CDN: Bunny.net

After analyzing application behavior and infrastructure bottlenecks, I identified the main performance issues and implemented the following improvements:

  • Private page caching: Enabled fast load times for returning users, delivering an immediate performance gain.
  • Lazy loading: Shifted part of the rendering workload to the frontend, reducing server-side processing and improving perceived load time.
  • Public page caching: After introducing lazy loading, pages could be safely cached publicly while private data was fetched asynchronously on the client side.
  • On-demand plugin loading: Modified WordPress initialization to load only required plugins per request, significantly improving API response times.
  • Cloud functions: Introduced serverless functions to deliver critical page data, improving resilience during peak traffic and server overload.

As a result, full page load time was reduced to approximately 2 seconds (a 4× improvement), while also significantly lowering overall server load and increasing platform stability.

Google Cloud Platform Migration

This project focused on the migration of the Fisarmonica learning platform from SaveInCloud to Google Cloud Platform, with the goal of improving scalability, reliability, and operational flexibility as the platform continued to grow.

The existing infrastructure presented limitations in scalability, resilience, and deployment flexibility, especially during traffic peaks and platform updates. To address these challenges, I designed and led the migration to a cloud-native architecture on Google Cloud.

The new infrastructure was built using the following core components:

  • Google Kubernetes Engine (GKE): Containerized the application and deployed it on a managed Kubernetes cluster, enabling horizontal scaling, rolling updates, and improved operational stability.
  • Cloud SQL: Migrated the relational database to a managed service, improving reliability, backup management, and operational maintenance.
  • NFS: Implemented shared storage to support persistent data and media files across multiple application instances.
  • Cloud Functions: Implemented serverless functions to handle critical system endpoints, increasing resilience, isolating failure domains, and ensuring availability during high-load scenarios.

As a result of this migration, the platform achieved improved availability, easier scalability, and a more robust deployment process. The cloud-native architecture provided a strong foundation for future growth, performance optimization, and continued feature development.

Web Platform UX and Visual Enhancement

This project focused on the visual enhancement and user experience improvement of an existing web platform. I was responsible for implementing the interfaces designed by the web designer and proposing improvements to increase usability, consistency, and overall user satisfaction.

Our main goal was to significantly improve the user experience by refining visual elements, navigation flow, and interaction patterns across the platform.

To better understand how users interacted with the application, we leveraged analytics and behavior tools such as Microsoft Clarity and Hotjar. These insights allowed us to identify usability friction points, analyze user journeys, and propose data-driven UX enhancements.

Key contributions included:

  • Implementation of responsive and accessible UI components based on the designer’s specifications, ensuring visual consistency across desktop and mobile devices.
  • UX improvements driven by user behavior analysis, including layout refinements, clearer information hierarchy, and optimized navigation flows.
  • Active collaboration with design, product, and business teams to propose and validate interface enhancements.
  • Introduction of gamification features such as points accumulation, achievement badges, and leaderboards to increase user engagement and retention.

As a result of these visual and UX improvements, the platform became more intuitive and engaging. Combined with the efforts of the sales team, these changes helped almost double the user base, directly supporting business growth and customer acquisition.
