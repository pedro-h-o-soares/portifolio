Web Platform Performance Optimization

The application originally took up to 8 seconds to fully load each page, which resulted in frequent user complaints and poor perceived performance. After internal discussions, I was responsible for investigating the root causes and implementing technical improvements.

The platform stack at the time was:

  • Framework: WordPress (PHP)
  • Database: MariaDB
  • Cloud provider: SaveInCloud
  • CDN: Bunny.net

After analyzing application behavior and infrastructure bottlenecks, I identified the main performance issues and implemented the following improvements:

  • Private page caching: Enabled fast load times for returning users, delivering an immediate performance gain.
  • Lazy loading: Shifted part of the rendering workload to the frontend, reducing server-side processing and improving perceived load time.
  • Public page caching: After introducing lazy loading, pages could be safely cached publicly while private data was fetched asynchronously on the client side.
  • On-demand plugin loading: Modified WordPress initialization to load only required plugins per request, significantly improving API response times.
  • Cloud functions: Introduced serverless functions to deliver critical page data, improving resilience during peak traffic and server overload.

As a result, full page load time was reduced to approximately 2 seconds (a 4× improvement), while also significantly lowering overall server load and increasing platform stability.

Web Platform UX and Visual Enhancement

This project focused on the visual enhancement and user experience improvement of an existing web platform. I was responsible for implementing the interfaces designed by the web designer and proposing improvements to increase usability, consistency, and overall user satisfaction.

Our main goal was to significantly improve the user experience by refining visual elements, navigation flow, and interaction patterns across the platform.

To better understand how users interacted with the application, we leveraged analytics and behavior tools such as Microsoft Clarity and Hotjar. These insights allowed us to identify usability friction points, analyze user journeys, and propose data-driven UX enhancements.

Key contributions included:

  • Implementation of responsive and accessible UI components based on the designer’s specifications, ensuring visual consistency across desktop and mobile devices.
  • UX improvements driven by user behavior analysis, including layout refinements, clearer information hierarchy, and optimized navigation flows.
  • Active collaboration with design, product, and business teams to propose and validate interface enhancements.
  • Introduction of gamification features such as points accumulation, achievement badges, and leaderboards to increase user engagement and retention.

As a result of these visual and UX improvements, the platform became more intuitive and engaging. Combined with the efforts of the sales team, these changes helped almost double the user base, directly supporting business growth and customer acquisition.
